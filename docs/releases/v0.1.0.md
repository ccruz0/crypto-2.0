# Release v0.1.0

- Date: `2026-03-07`
- Task ID: `implementation-bootstrap`
- Scope: Agent workflow versioning and traceability foundation

## Included Change

Implemented explicit version metadata lifecycle for OpenClaw task proposals:

- Proposal stage now carries `current_version`, `proposed_version`, `version_status`, and `change_summary`.
- Approval stage can persist `version_status=approved` or `version_status=rejected` in Notion when fields exist.
- Execution success marks released version metadata and records a release entry in the changelog.
